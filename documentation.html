Satvic Skin

Comprehensive Project Documentation & Step-by-Step Implementation Guide

Project Overview

Satvic Skin is a high-performance, responsive, and SEO-optimized one-page e-commerce website designed for a natural skincare brand. The project emphasizes a clean, premium aesthetic using a pastel green and off-white color palette, ensuring a calming user experience aligned with the brand's organic philosophy.

Tech Stack: HTML5, CSS3 (Vanilla), JavaScript (Vanilla).

2 Semantic HTML & SEO

We structured the document using semantic HTML5 tags to ensure accessibility and excellent search engine optimization (SEO).

flowchart LR Body[Body] --> Header[header] Body --> Main[main] Body --> Footer[footer] Main --> Hero[section id='home'] Main --> Features[section class='features'] Main --> About[section id='about'] Main --> Shop[section id='shop'] Main --> Newsletter[section id='contact']

SEO Best Practices Implemented:

  • Descriptive meta tags for viewport and description.
  • JSON-LD Schema Markup: Added HealthAndBeautyBusiness structured data to help search engines understand the website's context.
  • Use of alt attributes on all images for screen readers and image search.
  • Logical heading hierarchy (H1 for main title, H2 for sections, H3 for sub-sections).

3 CSS Styling & Layouts

The styling is built purely with Vanilla CSS, leveraging modern features like CSS Variables (Custom Properties), Flexbox, and CSS Grid for a robust responsive design.

Layout Strategy

We combined Grid and Flexbox where each shines best:

pie title "CSS Layout Methods Used" "CSS Grid (Product Cards, Footer, Hero)" : 60 "Flexbox (Navigation, Centering, Badges)" : 40
  • CSS Variables: Stored at the :root level for easy theming and consistency.
  • Responsive Grid: The product showcase uses grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); to automatically wrap and resize product cards based on screen width without media queries.
  • Glassmorphism: The sticky navigation header uses backdrop-filter: blur(10px) for a modern, premium feel as it scrolls over content.

4 JavaScript Interactivity

Vanilla JavaScript was used to add life to the website, focusing on performance without heavy libraries.

Core Functionalities

sequenceDiagram participant User participant Browser participant DOM User->>Browser: Scrolls down the page Browser->>DOM: Triggers 'scroll' event DOM-->>Browser: Checks scroll position Browser->>DOM: Adds 'scrolled' class to Header (Shadow) Browser->>DOM: IntersectionObserver detects elements DOM-->>Browser: Elements enter viewport Browser->>DOM: Adds 'appear' class (Fade-in Animation) Browser->>DOM: Updates Active Nav Link based on section
  • Intersection Observer: Used for performant scroll animations. Elements with the .fade-in class gracefully slide up and fade in as they enter the viewport.
  • Dynamic Active Links: As the user scrolls, the navigation links highlight based on which section is currently in view.
  • Shopping Cart Simulation: Clicking "Add to Cart" triggers visual feedback on the button and increments the cart counter with a subtle pulse animation.
  • Mobile Menu: A responsive hamburger menu that toggles the navigation links on smaller screens.

5 Animations & Polish

To achieve the "WOW" factor and a premium feel, micro-interactions were added throughout the UI.

  • Hover States: Buttons fill with color, product images scale up slightly (transform: scale(1.05)), and product cards elevate with increased shadows.
  • Staggered Delays: When a section comes into view, its elements don't appear all at once. By using .delay-1 and .delay-2 CSS classes, elements appear sequentially for a cascading effect.
  • Organic Shapes: A CSS-generated blob using complex border-radius values was placed behind the 'About' section image to reinforce the "natural" theme.
